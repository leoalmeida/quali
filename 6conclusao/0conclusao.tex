\chapter{Conclusão}\label{cap:conclusao}
Como pôde ser observado na se\c{c}\~{a}o destinada aos resultados, utilizar uma plataforma de IoT para automatiza\c{c}\~{a}o de métodos agrícolas existentes pode reduzir consideravelmente o trabalho de coleta e análise em tempo real da situação do meio ambiente utilizado para platio, como já era esperado.

A simula\c{c}\~{a}o utilizou uma implementa\c{c}\~{a}o da API apresentada neste trabalho para permitir que os dispositivos se comunicassem. Nestas simula\c{c}\~{o}es, dispositivos possuem uma aplica\c{c}\~{a}o mo\'{o}vel que cont\'{e}m dois servi\c{c}os, file-transfer e information-exchanger, conforme descritos neste trabalho. Estes servi\c{c}os s\~{a}o oferecidos atrav\'{e}s do Middleware que utiliza a simplifica\c{c}\~{a}o para identifica\c{c}\~{a}o de servi\c{c}os descrita.
